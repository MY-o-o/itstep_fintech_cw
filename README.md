# Класна робота. Proof of Work та оптимізація обчислень

## Завдання 1. Майнінг префікса (Vanity Hash)
**Мета:** Змусити систему знайти хеш, який починається з певного слова у шістнадцятковому форматі. Оскільки хеш складається з символів 0-9 та a-f, можна шукати слова на кшталт cafe, bad, c0de.

**Що і де потрібно змінити:**

1. У файлі MiningService.cs змініть параметри методу MineBlock. Замість int difficulty він має приймати string targetPrefix.
2. У цьому ж методі видаліть генерацію рядка з нулів.
3. Змініть умову виходу з циклу: замість block.Hash.StartsWith(target) напишіть block.Hash.StartsWith(targetPrefix).
4. У файлі Program.cs створіть блок, запишіть у поле Data ваше прізвище і запустіть майнінг з префіксом "cafe".
5. Зафіксуйте кількість спроб (Nonce), яка знадобилася для знаходження такого хешу.

## Завдання 2. Вимірювач швидкості (Hashrate Benchmark)
**Мета:** Написати логіку, яка буде в реальному часі показувати, скільки хешів за секунду (H/s) обчислює процесор комп'ютера.

**Що і де потрібно змінити:**

1. У файлі MiningService.cs всередині методу MineBlock створіть об'єкт Stopwatch перед початком циклу while.
2. Усередині циклу додайте умову: якщо кількість спроб (attempts) ділиться без остачі на 50000, програма має розрахувати поточну швидкість.
3. Формула розрахунку: розділіть кількість спроб на кількість секунд, що пройшли (stopwatch.Elapsed.TotalSeconds).
4. Виведіть результат у консоль за допомогою Console.Write("\rШвидкість: ... H/s"). Використання \r дозволить оновлювати дані в одному рядку консолі, не створюючи нових.

## Завдання 3. Багатопотоковий майнінг (Складний рівень)
**Мета:** Зараз програма використовує лише одне ядро процесора. Завдання полягає в тому, щоб переписати логіку перебору Nonce для використання всіх доступних ядер комп'ютера, що суттєво пришвидшить майнінг на високій складності.

**Що і де потрібно змінити:**

1. У файлі MiningService.cs повністю перепишіть метод MineBlock.
2. Замість звичайного циклу while використайте клас Parallel.
3. Налаштуйте логіку так, щоб різні потоки не перевіряли одні й ті самі значення Nonce. Наприклад, можна дати кожному потоку власний крок (один потік перевіряє тільки парні числа, інший - непарні, або розділити їх по діапазонах).
4. Додайте механізм скасування. Використайте ParallelLoopState.Stop() або CancellationToken. Коли один із потоків знаходить правильний хеш, він має записати його в блок і миттєво зупинити роботу всіх інших потоків.
