# Просунуте завдання на 12 балів: Математична складність (BigInteger Target)

> На лекціях ми розбирали, що складність (Difficulty) виражається у кількості початкових нулів у хеші. Це абсолютно правильне візуальне пояснення, яке використовується в усіх блокчейн-експлорерах (навіть для Bitcoin), щоб людям було зручно оцінювати складність очима.
> Але "під капотом", на рівні коду ядра, комп'ютери не рахують нулі в рядках — це занадто повільно і не дозволяє плавно регулювати час. Натомість використовується концепція Target — гігантського числа. Початкові нулі з'являються саме тому, що згенерований хеш (як число) має бути меншим за цей Target. Що менший Target, то більше нулів з'являється на початку шістнадцяткового запису.

### **Ваше завдання:** Перевести нашу систему з "візуального" підрахунку нулів на справжню криптографічну математику, використовуючи структуру BigInteger з простору імен System.Numerics.

**Що потрібно зробити:**

1. Заміна логіки перевірки: Відмовтеся від методу перевірки рядка на нулі. Тепер ваш метод майнінгу повинен брати згенерований хеш, конвертувати його в об'єкт BigInteger і перевіряти математичну умову: згенерований хеш < Target.
2. Плавний перерахунок (Adjust Difficulty): Перехід на Target дозволяє регулювати складність дуже плавно (а не стрибками у 16 разів при додаванні одного нуля). Змініть логіку динамічного коригування. Формула: Новий Target = Старий Target \* (Реальний час / Очікуваний час). Якщо майнили занадто швидко, число Target зменшується.
3. Обмеження меж (Clamp): Напишіть логіку, яка не дозволить Target змінюватися більше ніж у 4 рази за один перерахунок, щоб захистити мережу від різких стрибків.
4. Оновлення виводу: Оновіть відображення статистики в консолі. Ви можете залишити вивід кількості нулів для наочності, але обов'язково додайте вивід поточної складності як відсоток від початкового Target або у вигляді експоненціального числа.
5. Тест-драйв: Запустіть програму з цільовим часом 5 секунд на блок і переконайтеся, що система тепер коригує час дуже плавно і точно, а не стрибає хаотично від нуля до нескінченності.
